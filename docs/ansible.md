# Ansible Operations

Ansible handles reconciliation after Terraform converges infrastructure state.

## Playbooks

- `ansible/playbooks/reconcile-vpn-nodes.yml`
  - consumes `ansible/inventory/generated/vpn_nodes.yml` (generated by Terraform)
  - reconciles WireGuard peers and Swarm membership/labels
  - decommissions disabled or removed nodes

- `ansible/playbooks/deploy-stacks.yml`
  - validates foundation dependencies
  - deploys stacks (`traefik`, `monitoring`, `swarmpit`, `vpn`, optional `vpn-dev`, optional `probe`)

## Inputs

Topology is not read directly from env JSON. It is read from generated inventory files:
- `ansible/inventory/generated/vpn_nodes.yml`
- `ansible/inventory/generated/infra_nodes.yml` (for infra workflows)

Environment variables are used for stack toggles/secrets/version pins, for example:
- `ENABLE_VPN_DEV_STACK`
- `DEPLOY_PROBE_STACK`
- `RUN_SANITY_CHECK`
- `VPN_DOMAIN`, `VPN_WS_PATH`, `VPN_XHTTP_PATH`
- version pins: `PROMETHEUS_CONFIG_VERSION`, `XRAY_CONFIG_VERSION`, etc.

## Run

```bash
set -a
source .env
set +a

export REPO_ROOT="$(pwd)"
export ANSIBLE_CONFIG="$(pwd)/ansible/ansible.cfg"

ansible-playbook -i ansible/inventory/production.ini ansible/playbooks/reconcile-vpn-nodes.yml
ansible-playbook -i ansible/inventory/production.ini ansible/playbooks/deploy-stacks.yml
```

## SSH host keys

`ansible.cfg` uses strict host key checking.
Keep target host keys in known_hosts on the runner/manager.
